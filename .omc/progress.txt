Learnings for next ralph session on this repo:
- mkdocs-material does not use a `not_found:` config key; the 404.md file in docs/
  is enough. Don't add `not_found:` to mkdocs.yml.
- For files in docs/ that should not be published (working notes, allowlists), use
  `exclude_docs: |\n  _*.md` in mkdocs.yml. mkdocs 1.6+ accepts gitignore-style patterns.
- Material theme overrides live in `overrides/` (set via theme.custom_dir). Jinja in
  these files does NOT trigger the docs/ Jinja grep gate — they're not content.
